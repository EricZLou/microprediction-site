Why MUIDs?

You need a write_key to use the microprediction APIs or python library. We use MUID.org (stands for Memorable Unique Identifiers) as a source of write_keys that take some time to generate. There is an explanatory video.

MUID write keys are "heavy" and that's your way of signalling that you are serious. Hopefully you have found or engineered a live time series that is important to someone, and you intend to keep publishing it indefinitely. Streams that aren't maintained get garbage collected pretty quickly, but we'd prefer not to have spurious streams to begin with. We also require those submitting predictions to possess a MUID. This discourages the creation of a large number of noisy prediction algorithms.

Minimum key difficulty

The length of the memorable part of the hash of the key is known as the difficulty. See the video. To create a stream (see crowd) you will require a write_key of difficulty {{min_len}}. To supply predictions the bar is lower and currently set at a minimum difficulty of 8.

Bankruptcy

All write_keys have an associated balance. See the microprediction package README for a table of minimum allowed balances. A write_key whose balance falls below these thresholds can no longer be used.

How to get a key

Your choices are
  1. Click this link to get a key of borderline difficulty 8.
  2. Run this bash command in terminal.
  3. Pip install the muid package and run muid.mine()
For convenience we copy choice 2 here: 
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/microprediction/muid/master/examples/mine_from_venv.sh)"
It may take a day or so to produce a MUID with difficulty {{min_len}}. If it doesn't work then perhaps you need Python 3 and/or a package manager: 
    ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"
    brew install python
then return to the mining script.
