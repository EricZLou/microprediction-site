Memorable Unique Identifiers
We created the concept of a memorable unique identifier (and MUID.org) in order to:
  • Keep things democratic
  • Have one less join, because the sha256 hash of your write_key is your public identity,
  • Discourage spurious streams, and
  • Discourage noisy predictions and crawlers.

By construction Memorable Unique Identifiers are rare and that's your way of signalling that you are serious. Hopefully you have found or engineered a live time series that is important to someone, and you intend to keep publishing it indefinitely. Streams that aren't maintained get garbage collected pretty quickly, but we'd prefer not to have spurious streams to begin with.

Minimum key difficulty settings
The length of the memorable part of the hash of the key (such as Losable Flea) is known as the difficulty.
  • To create a stream, you will require a write_key of difficulty {{min_len}}.
  • To create a copula stream (joint distributions) you will require a write_key of difficulty {{min_len}}+1.
  • To supply predictions the bar is lower and currently set at a minimum difficulty of {{min_len}}-4.
If you mine a better nom de plume than Foldable Cat then by all means let everyone know.
Bankruptcy
All write_keys have an associated balance. See the microprediction package README for a table of minimum allowed balances. A write_key whose balance falls below these thresholds can no longer be used.
How to get a MUID to use as a write_key
Your choices are:
  1. Click this link to get a key of borderline difficulty 8.
  2. Run this bash command in terminal.
  3. Pip install the muid package and run muid.mine()
  4. Beg. Email info@microprediction.org and explain your intended use
Nice people have been donating their CPU time to mine this stockpile of MUIDs intended for civic, educational or scientific purposes.
Start mining MUIDs with a cut and paste (mac/linux only)
For convenience we copy choice 2 here: 
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/microprediction/muid/master/examples/mine_from_venv.sh)"
It may take a day or so to produce a MUID with difficulty {{min_len}}. If it doesn't work then perhaps you need Python 3 and/or a package manager: 
    ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"
    brew install python
then return to the mining script.
